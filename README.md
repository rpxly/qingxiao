## 二、素材管理建议

### 1. 背景图片处理

#### 推荐方案：
- **古代背景图**：
  - 方案1：使用《剑网3》截图+PS滤镜处理（推荐）
  - 方案2：使用AI生成的古风场景（如DALL-E、Midjourney）
  - 方案3：使用免费可商用的古风素材网站资源
- **现代背景图**：
  - 方案1：使用Unsplash都市夜景图片+模糊处理（推荐）
  - 方案2：使用城市建筑摄影作品
  - 方案3：使用AI生成的现代都市场景

#### 图片优化：
- **尺寸标准**：建议1920x1080像素（16:9比例）
- **格式选择**：
  - 静态图片：使用JPG格式，压缩质量70-80%
  - 透明图片：使用PNG格式
- **大小控制**：单个图片控制在200KB以内
- **工具推荐**：
  - 图片压缩：TinyPNG、Squoosh
  - 批量处理：Photoshop批处理、ImageMagick

#### 替换方法：
1. **准备素材**：将优化后的背景图片保存到 `assets/images/` 文件夹
2. **修改CSS**：在 `css/style.css` 文件中更新背景图片URL：
   ```css
   .ancient {
       background: linear-gradient(rgba(0, 0, 0, 0.7), rgba(0, 0, 0, 0.7)), url('assets/images/ancient-bg.jpg');
   }
   .modern {
       background: linear-gradient(rgba(0, 0, 0, 0.7), rgba(0, 0, 0, 0.7)), url('assets/images/modern-bg.jpg');
   }
   ```
3. **测试效果**：在不同设备上测试背景图片的显示效果

#### 最佳实践：
- 使用相对路径而非绝对路径
- 为背景图片添加适当的渐变叠加，确保文字可读性
- 考虑添加图片预加载功能，提升用户体验

### 2. 团队头像制作

#### 推荐方案：
- **AI生成**：
  - 方案1：使用Midjourney生成古风Q版头像（推荐）
  - 方案2：使用DALL-E 3生成风格统一的头像
  - 方案3：使用国内AI工具如百度文心一言、讯飞星火
- **手绘扫描**：手绘团队成员头像后扫描成电子版，使用PS处理
- **照片处理**：使用团队成员照片，添加古风滤镜处理（如醒图、美图秀秀）

#### 头像规格：
- **尺寸**：200x200像素（正方形）
- **格式**：PNG格式（支持透明背景）
- **风格**：统一的古风Q版风格，保持视觉一致性
- **分辨率**：72DPI（网页显示足够）
- **大小**：单个头像控制在50KB以内

#### 替换方法：
1. **准备素材**：将制作好的头像保存到 `assets/images/team/` 文件夹
2. **修改HTML**：在 `index.html` 文件中更新团队头像部分：
   ```html
   <div class="member">
       <div class="member-avatar" style="background-image: url('assets/images/team/fufu.jpg'); background-size: cover;"></div>
       <h4>孚孚</h4>
       <p>编剧+分镜+演员（之一）</p>
   </div>
   ```
3. **测试显示**：在不同设备上测试头像显示效果

#### 命名规范：
- 使用团队成员昵称作为文件名（小写）
- 例如：`fufu.jpg`、`xiaoya.jpg`、`zhangyu.jpg`等
- 保持文件名的一致性和可读性

### 3. 进度数据管理

#### 数据文件结构：
`assets/data/progress.json` 文件结构：
```json
{
  "milestones": [
    {
      "title": "初稿进度",
      "percent": "65%"
    },
    {
      "title": "剧本修改",
      "percent": "40%"
    },
    {
      "title": "视觉设计",
      "percent": "25%"
    }
  ],
  "gallery": [
    {
      "id": "1",
      "title": "拍摄现场",
      "description": "古代场景拍摄花絮",
      "image": "图片URL",
      "category": "拍摄花絮",
      "date": "2026-01-20"
    }
  ]
}
```

#### 数据更新方法：
1. **修改进度数据**：直接编辑 `assets/data/progress.json` 文件中的 `milestones` 数组
2. **添加新进度**：在 `milestones` 数组中添加新的进度项
3. **删除进度**：从 `milestones` 数组中删除不需要的进度项
4. **更新剧照花絮**：编辑 `assets/data/progress.json` 文件中的 `gallery` 数组
5. **添加新剧照**：在 `gallery` 数组中添加新的剧照花絮项
6. **删除剧照**：从 `gallery` 数组中删除不需要的剧照花絮项

#### 实现原理：
- 项目已在 `js/script.js` 文件中实现了两个函数：
  - `loadProgressData()`：加载和显示进度数据
  - `loadGalleryData()`：加载和显示剧照花絮数据
- 两个函数都通过 `fetch` API 从 JSON 文件加载数据
- 在里程碑区域下方动态创建对应容器和内容
- 无需修改HTML或JavaScript代码，只需更新JSON文件即可

#### 数据格式说明：
- **milestones 数组**：
  - **title**：进度项名称（字符串）
  - **percent**：完成百分比（字符串，格式为 "XX%"）
  - 支持添加任意数量的进度项

- **gallery 数组**：
  - **id**：唯一标识符（字符串）
  - **title**：剧照标题（字符串）
  - **description**：剧照描述（字符串）
  - **image**：图片URL（字符串）
  - **category**：分类（字符串，如"拍摄花絮"、"幕后制作"等）
  - **date**：日期（字符串，格式为 "YYYY-MM-DD"）
  - 支持添加任意数量的剧照花絮项

#### 最佳实践：
- 定期更新进度数据，保持项目信息的时效性
- 使用有意义的进度项名称，便于团队理解
- 确保百分比数据真实反映项目实际进展

## 五、常见问题解决

### 1. 图片加载缓慢

#### 问题原因：
- 图片文件过大
- 服务器响应慢
- 网络连接问题

#### 解决方案：
- **图片优化**：
  - 使用TinyPNG或Squoosh压缩图片
  - 选择合适的图片格式（JPG for 照片，PNG for 透明背景）
  - 实现图片懒加载
- **预加载关键图片**：
  - 对首屏图片使用预加载
  - 考虑使用WebP格式（现代浏览器支持）

### 2. 跨设备显示问题

#### 问题原因：
- 响应式设计未正确实现
- 不同浏览器渲染差异
- 设备屏幕尺寸多样

#### 解决方案：
- **响应式设计**：
  - 确保CSS媒体查询正确设置
  - 使用相对单位（rem, em, %）而非固定像素
  - 测试不同屏幕尺寸
- **浏览器兼容性**：
  - 测试主流浏览器（Chrome, Firefox, Safari, Edge）
  - 使用CSS前缀处理兼容性问题
  - 考虑使用PostCSS自动处理兼容性
- **调试工具**：
  - 使用浏览器开发者工具的设备模拟器
  - 使用在线测试工具如BrowserStack

### 3. 素材版权问题

#### 问题原因：
- 使用了有版权的素材
- 未获得素材使用许可
- 素材来源不明

#### 解决方案：
- **使用免费可商用素材**：
  - 图片：Unsplash, Pexels, Pixabay
  - 字体：Google Fonts, 站酷字体
  - 图标：Font Awesome, Material Icons
- **AI生成素材**：
  - 使用Midjourney, DALL-E等生成原创素材
  - 确保AI工具的使用条款允许商业使用
- **版权声明**：
  - 对于需要注明来源的素材，添加适当的版权声明
  - 保留素材的原始许可证信息

### 4. 数据加载失败

#### 问题原因：
- JSON文件格式错误
- 文件路径不正确
- 跨域访问问题

#### 解决方案：
- **JSON格式验证**：
  - 使用JSON验证工具检查格式
  - 确保所有字符串使用双引号
  - 检查逗号和括号是否正确
- **路径检查**：
  - 确保使用相对路径
  - 检查文件名大小写是否正确
  - 验证文件是否存在于指定位置
- **跨域处理**：
  - 本地测试时使用本地服务器

### 5. 性能优化问题

#### 问题原因：
- CSS和JavaScript文件过大
- 过多的HTTP请求
- 未优化的动画效果

#### 解决方案：
- **代码优化**：
  - 压缩CSS和JavaScript文件
  - 合并多个CSS/JS文件
  - 移除未使用的代码
- **资源加载**：
  - 使用适当的资源加载策略（preload, prefetch）
  - 减少第三方脚本的使用
  - 优化字体加载
- **动画性能**：
  - 使用CSS transform和opacity进行动画
  - 避免频繁的DOM操作
  - 考虑使用requestAnimationFrame
